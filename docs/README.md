## Contributing to Documentation

Documentation improvements are welcome!

To contribute:

1. Review [../CONTRIBUTING.md](../.github/CONTRIBUTING.md)
2. Improve existing documentation or write new guides
3. Submit your changes via GitHub pull request
4. Include clear descriptions of changes

Documentation should be:
- Clear and accessible to non-specialists
- Well-organized with clear headings
- Inclusive of diverse perspectives
- Accurate and up-to-date
- Linked to relevant resources

## Versioning

Documentation versions generally align with project versions:
- Major changes reflected in section updates
- Ongoing improvements documented in GitHub history
- Archives of previous versions maintained
